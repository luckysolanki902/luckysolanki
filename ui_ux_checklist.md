# Universal UI/UX Design Principles  
*A checklist to think through before designing any website or product.*

---

# 1. Clarity Over Cleverness

If a user has to think about what something means, the design has failed.

- Use familiar patterns.
- Label clearly.
- Avoid novelty for novelty’s sake.
- Remove decorative confusion.

> If it needs explanation, redesign it.

---

# 2. Define the Primary Action

Every screen must answer:

**What is the one thing the user should do here?**

- One primary CTA.
- Clear visual dominance.
- Secondary actions should never compete.

If everything is important, nothing is important.

---

# 3. Reduce Cognitive Load

Human working memory is limited (~4 items at once).

- Limit choices.
- Break content into chunks.
- Use whitespace generously.
- Avoid information overload above the fold.

Design for mental ease.

---

# 4. Visual Hierarchy Controls Attention

Users scan, they don’t read.

Use:
- Size
- Weight
- Contrast
- Spacing
- Position

Hierarchy should guide the eye naturally:
1. Headline
2. Subtext
3. Primary action
4. Supporting detail

If users don’t know where to look first, hierarchy is broken.

---

# 5. Consistency Builds Trust

Consistency reduces cognitive friction.

Be consistent in:
- Spacing scale
- Typography scale
- Color usage
- Button styles
- Interaction patterns

A system is stronger than a collection of components.

---

# 6. Proximity Creates Meaning (Gestalt Principle)

Related items belong close together.  
Unrelated items should have space between them.

Spacing is not decoration.  
Spacing communicates relationships.

---

# 7. Minimize Decisions (Hick’s Law)

More choices = slower decisions.

- Limit navigation items.
- Avoid dropdown overload.
- Keep forms minimal.
- Reduce optional branches.

Clarity accelerates action.

---

# 8. Make Important Targets Easy to Click (Fitts’s Law)

The time to click depends on size and distance.

- Large tap targets (minimum 44×44px).
- Keep primary actions within easy reach.
- Avoid tiny links in dense text.

Usability is physical, not just visual.

---

# 9. Design for Scanning (Not Reading)

Users:
- Scan headlines
- Scan bold text
- Scan bullets
- Scan images

Use:
- Short paragraphs
- Subheadings
- Bullet lists
- Visual separation

Dense walls of text reduce engagement.

---

# 10. Whitespace Is Functional

Whitespace:
- Improves readability
- Increases perceived value
- Reduces anxiety
- Signals confidence

When in doubt:
Add more space.

---

# 11. Feedback Is Mandatory

Every user action must produce visible feedback.

Examples:
- Button hover states
- Loading indicators
- Form validation messages
- Success confirmations

No feedback = user uncertainty.

---

# 12. Motion Must Communicate

Animation should answer:
- What changed?
- Where did this come from?
- What should I look at?

Never animate purely for decoration.

If it doesn’t clarify, remove it.

---

# 13. Progressive Disclosure

Reveal complexity gradually.

Surface: simple  
Depth: detailed  

Don’t overwhelm users on entry.  
Let them explore deeper layers intentionally.

---

# 14. Accessibility Is Not Optional

- Minimum 4.5:1 contrast ratio for body text
- Keyboard navigable interface
- Visible focus states
- Semantic HTML structure
- Respect `prefers-reduced-motion`

Inclusive design is good design.

---

# 15. Performance Is Part of UX

Speed affects perception of quality.

- Optimize images
- Avoid heavy libraries
- Lazy-load non-critical content
- Keep interaction latency low

Fast feels professional.

Slow feels careless.

---

# 16. One Visual Idea Per Screen

Avoid competing design ideas.

Each page should have:
- One dominant headline
- One visual focal point
- One main action

Design is subtraction.

---

# 17. Align Form With Function

Visual tone must match purpose.

Financial product → stable, calm  
Creative studio → expressive, bold  
Developer portfolio → precise, confident  

Aesthetic must reinforce intent.

---

# 18. The Peak-End Rule

Users remember:
- The first impression
- The final interaction

Make:
- Hero section intentional
- Closing section emotionally satisfying

End strong.

---

# 19. Measure, Don’t Assume

Good design is validated.

- Observe behavior
- Track clicks
- Analyze drop-offs
- Conduct usability tests

Opinion ≠ evidence.

---

# 20. Remove Before You Add

Before adding an element, ask:
- What happens if this doesn’t exist?
- Does this increase clarity?
- Does this increase conversion?
- Does this increase trust?

If not — remove it.

---

# Final Rule

Good UI feels invisible.  
Good UX feels inevitable.

When a product feels obvious, intuitive, and calm —  
that’s not simplicity by accident.

That’s design done right.
